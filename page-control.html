Control de dispositivos

La gestión de periféricos y componentes internos es fundamental en cuaquier sisteme operativo. Aprende cómo el kernel interactúa con el hardware a través de controladores y archivos de dispositivos especiales.

Driver (Controladores)

Los controladores actúan como traductores entre el sistema operativo y el hardware. Sin ellos el kernel no sabría cómo enviar comandos especificos a una impresora o tarjeta grafica. En sistemas modernos, muchos vienen integrados directamente.

/dev es el directorio en Linux que contiene archivos especiales para acceder a los dispositivos de hardware.

udev es el gestor de dispositivos que crea o elimina estos archivos dinámicamente en /dev, gestiona eventos de "hotplug" (conectar/desconectar hardware) y carga controladores. Permite reglas personalizadas para nombrar y asignar permisos a dispositivos.

>_ Ejemplos de comandos

Listar dispositivos usb

lsusb.

Ver dispositivos de bloque

lsblk.

Informacion del kernel

dmesg | tail -n20

Listar buses PCI

lspci -v

Regla udev personalziada

Ejemplo para montar automaticamente un disco espesifico:

/etc/udev/rules.d/99-custom.rules

KERNEL=="sdb1", SYMLINK+="backup_drive", MODE="0660", GROUP="storage"
